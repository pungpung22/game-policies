# Ghost Paws 동의 및 SDK 변경 체크리스트

## 현재 빌드

현재 빌드는 개인정보를 기기 밖으로 전송하지 않고 광고·추적 SDK가 없으므로 광고 동의 플랫폼이나 iOS ATT 프롬프트를 표시하지 않는다. 불필요한 동의 프롬프트를 추가하지 않는다.

앱 안에서는 다음 정보를 쉽게 확인할 수 있어야 한다.

- 개인정보를 수집·전송하지 않는다는 현재 상태
- 진행 정보가 기기에만 저장된다는 사실
- 앱/저장공간 삭제 방법
- 공개 개인정보처리방침 및 지원 URL

## 광고 SDK 추가 전

1. 광고 네트워크, mediation adapter와 모든 간접 SDK 버전을 확정한다.
2. IP 주소, 광고 ID, 상호작용, 진단, 대략적 위치 등 처리 항목과 목적을 기록한다.
3. 지역별 동의 요구사항을 검토하고 필요한 경우 Google UMP 등 동의 플랫폼을 SDK 초기화 전에 실행한다.
4. iOS에서 다른 회사 앱/웹을 가로질러 추적하는 경우에만 ATT 설명과 프롬프트를 적절한 시점에 제공한다.
5. 동의 거부 시 비개인화 광고 또는 광고 없는 대체 흐름이 정상인지 확인한다.
6. 보상은 광고 SDK의 공식 earned-reward 콜백 뒤 한 번만 지급한다.
7. 개인정보처리방침, Apple App Privacy, Google Data safety와 Ads declaration을 갱신한다.
8. 아동 또는 혼합 연령 대상이면 Families/아동 대상 광고 요구사항을 별도로 검토한다.

## 분석·충돌 보고·계정 추가 전

- 앱 시작 전에 자동 전송되는 항목을 포함해 SDK 기본값을 확인한다.
- 수집 목적, 보유 기간, 공유 대상, 이용자 선택과 삭제 방법을 문서화한다.
- 필요한 경우 앱 내 명확한 고지와 동의를 데이터 전송 전에 제공한다.
- 계정 생성이 생기면 앱 내부 계정 삭제 기능과 공개 삭제 안내를 준비한다.

## 릴리스 승인 게이트

- 최종 바이너리 SDK 목록과 정책 문서가 일치한다.
- 테스트 광고 ID와 실제 광고 ID가 환경에 맞다.
- 동의 이전 네트워크 요청이 없는지 실제 기기에서 확인한다.
- ATT 거부, 동의 거부, 오프라인, 광고 실패·닫기, 중복 콜백을 테스트한다.
- 정책 URL이 HTTPS로 공개되고 로그인 없이 열린다.
