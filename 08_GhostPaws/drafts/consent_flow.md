# Ghost Paws 광고 동의 및 개인정보 선택 흐름

검토일: 2026년 7월 13일

## 현재 런타임 흐름

1. 앱 실행 후 Android/iOS AdMob native plugin과 릴리스 광고 단위 ID를 확인한다.
2. UMP consent information을 매 실행마다 갱신한다.
3. 동의가 필요하지 않거나 획득된 상태에서만 Google Mobile Ads를 초기화한다.
4. 동의가 필요하고 양식이 있으면 먼저 양식을 표시한다.
5. 상태가 불명확하거나 양식을 완료하지 못하면 광고를 요청하지 않는다.
6. Google이 개인정보 양식을 제공하는 경우 설정 및 상점에 `광고 개인정보 설정`
   버튼을 표시한다.
7. 상점의 사용자가 직접 버튼을 눌렀을 때만 preload된 보상 광고를 표시한다.
8. SDK의 reward-earned 콜백에서만 생선쿠키 20개를 한 번 지급·저장한다.
9. 광고 닫기/실패 뒤에는 현재 일일 한도를 확인하고 다음 광고를 preload한다.

## 실기기 테스트

- EEA 테스트 지역에서 첫 실행 동의 양식과 재선택 진입점
- 동의 허용/거부, 양식 닫기, 오프라인, 광고 로드·표시 실패
- reward 콜백 전 닫기 미지급, reward 콜백 중복 시 1회 지급
- 앱 강제 종료·재실행 후 쿠키와 UTC 일일 횟수 유지
- 하루 3회 이후 버튼 차단, 다음 UTC 일자 초기화, 과거 날짜 되돌리기 방어
- 디버그는 Google 테스트 ID, 릴리스 후보는 계정에 등록한 테스트 기기만 사용

## 현재 한계와 출시 게이트

Poing 플러그인 4.3.1의 UMP 래퍼는 최신 native API의 `canRequestAds`와
`privacyOptionsRequirementStatus`를 직접 노출하지 않고 consent status/form
availability를 사용한다. 출시 전에 플러그인의 최신 릴리스에서 이 API 지원 여부를
다시 확인하고, 지원 버전이 나오면 갱신하는 것을 권장한다.

iOS는 현재 ATT 프롬프트를 표시하지 않는다. 향후 다른 회사 앱·웹을 가로지르는
IDFA 기반 추적을 선택할 경우 ATT 설명, 요청 시점, 동의 전 광고 흐름, App Privacy를
먼저 설계·갱신한다.
