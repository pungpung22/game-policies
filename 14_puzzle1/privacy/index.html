개인정보처리방침

시행일 및 최종 업데이트: 2026년 7월 20일

pungpungstudi(이하 “개발자”)는 모바일 게임 Boom Booth(com.pungpungstudi.boombooth, 이하 “게임”)의 개인정보 처리 방식을 다음과 같이 안내합니다.

요약: 게임은 계정 가입을 요구하지 않으며 진행 데이터를 기기에 저장합니다. Firebase를 이용해 최소한의 사용 분석, 충돌 진단, 원격 설정 및 서버 시간 확인을 수행합니다. 광고 식별자 기반 추적은 사용하지 않습니다.

1. 계정과 기기 저장 데이터

게임은 이름, 이메일, 비밀번호를 직접 수집하지 않습니다. 스테이지 진행, 골드, 하트, 아이템 수량, 튜토리얼 상태, 하트 회복 기준 시각 및 소리·진동 설정은 기기에 저장됩니다. 앱을 삭제하거나 운영체제의 앱 데이터 삭제 기능을 사용하면 로컬 데이터가 삭제될 수 있습니다.

2. Firebase

게임은 Google Firebase Analytics, Crashlytics, Remote Config 및 Realtime Database의 서버 시간 정보를 사용합니다.

  • Analytics: 세션 시작, 스테이지 시도·결과, 상점 화면 열기, 게임 내 재화로 구매한 아이템 등 제품 상호작용을 집계합니다.
  • Crashlytics: 충돌 로그, 앱·운영체제·기기 정보와 마지막 게임 상태 같은 진단 정보를 처리할 수 있습니다.
  • Remote Config: 하트 회복 시간, 게임 밸런스 값과 선택·필수 업데이트 안내를 원격으로 제공합니다.
  • 서버 시간: 기기 시계 변경을 이용한 하트 회복 조작을 줄이기 위해 Firebase 서버 시간과의 차이를 확인합니다.

Firebase가 앱 인스턴스 식별자, IP 주소를 통한 대략적인 지역, 앱·운영체제·기기 및 네트워크 정보를 처리할 수 있습니다. 개발자는 Firebase에 이름, 이메일 또는 별도의 사용자 계정을 설정하지 않으며 광고 저장소, 광고 사용자 데이터 및 광고 개인화 동의 신호는 거부 상태로 유지합니다.

3. 구매와 광고

앱 내 상품이 제공되는 경우 결제는 Apple App Store 또는 Google Play가 처리하며 개발자는 전체 카드 정보를 직접 받지 않습니다. 상품 제공을 위해 상품 식별자, 구매 상태 및 거래 식별자가 스토어 서비스와 교환될 수 있습니다. 현재 빌드는 광고 식별자 기반 맞춤 광고나 이용자 추적을 사용하지 않습니다.

4. 보관, 삭제와 제3자

로컬 데이터는 앱 데이터 삭제 또는 앱 제거 시까지 기기에 남습니다. Firebase와 앱 스토어에 전송된 정보는 Google과 Apple의 보관 정책 및 관련 법령에 따라 처리됩니다. 게임은 카메라, 마이크, 연락처, 사진 보관함 또는 정밀 위치 권한을 요구하지 않습니다.

5. 아동, 보안 및 변경

게임은 만 13세 미만 아동의 개인정보를 의도적으로 수집하도록 설계되지 않았습니다. 개발자는 개인정보를 직접 판매하지 않습니다. 기능, SDK 또는 법률이 실질적으로 변경되면 본 방침과 스토어 신고를 갱신합니다.
